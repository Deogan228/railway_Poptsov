# Вариант 13. Железнодорожная касса

Лабораторная работа №1: Монады (Scala 3).

Касса продаёт билеты, учитывает багаж и свободные места.

## Запускк

sbt run

Для русских символов в консоли Windows предварительно:
chcp 65001

## Запуск тестов

sbt "Test / runMain RailwayTests"

## Управление

- Вводите номер пункта меню и нажмите Enter.
- `0` — возврат / выход из программы.

## Структура проекта

src/main/scala/
├── monads/      — реализации монад (Monad, IO, Reader, Writer, State)
├── domain/      — предметная область (типы, Reader/Writer/State функции)
├── plan/        — сценарий IO (действия меню)
└── ui/          — меню и цикл взаимодействия

## Что где используется

### Reader (`FuncReader.scala`)

Чтение конфигурации кассы:
- `ticketPrice(route, classType)` — цена билета по маршруту и классу
- `baggageCost(weight)` — доплата за багаж
- `seatAvailable(train, seat)` — свободно ли место
- `refundAmount(ticket)` — сумма возврата с учётом штрафа

### Writer (`FuncWriter.scala`)

Логирование событий: выбор маршрута, расчёт цены, назначение места, возврат.
Также используется внутри `FuncState` для логов переходов состояния.

### State (`FuncState.scala`)

Состояние: поезда с местами, проданные билеты, выручка, счётчик id.

Переходы:
- `bookTicket` — оформить билет
- `cancelTicket` — вернуть билет (с возвратом денег за вычетом штрафа)
- `addTrain` — добавить новый поезд
- `nextDay` — новый день (сброс билетов и выручки)

### IO (`Plan.scala`, `ui/`)

Взаимодействие через консольное меню: показать поезда, купить/вернуть билет,
добавить/удалить поезд, перейти к следующему дню.
